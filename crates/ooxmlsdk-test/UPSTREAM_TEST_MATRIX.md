## Coverage Rules

- Track upstream behavior through public Rust APIs, not one-to-one ports of .NET
  implementation tests.
- Keep package and part coverage aligned with upstream container concepts:
  package open/save, relationship graph traversal, add/delete/get part, supported
  relationship helpers, media/data part references, clone/copy/save, and part URI
  allocation.
- Keep schema, simple type, and validator coverage close to the behavior being
  protected, using upstream fixture XML and package assets where possible.
- Do not test raw package storage, generated child descriptors, relationship
  storage internals, or root cache state from this crate.
- If a behavior matters but only an internal API exposes it, assert it through
  package open/save/reopen, public part accessors, or serialized XML.
- Keep `// Source:` comments on upstream-derived tests so new coverage can be
  audited against `../Open-XML-SDK/test`.
