# VBA Macro Preservation — ooxmlsdk Clean-Room Spec

**Source authority:** ECMA-376 5th edition Part 1 §22.5 (VBA project part);
ISO/IEC 29500:2016 Part 1 §22.5; MS-OVBA (Microsoft Open Specifications).

---

## 1. Overview

VBA macros are stored as an OLE2 compound document binary (`vbaProject.bin`)
inside macro-enabled OOXML packages. ooxmlsdk treats this part as an **opaque
binary blob** — it is read, preserved, and written back unchanged with no
interpretation of the OLE2 structure or VBA bytecode.

Macro-enabled variants:

| Format | Extension | Root content type |
|--------|-----------|-------------------|
| Word macro-enabled | `.docm` | `…wordprocessingml.document.macroEnabled.main+xml` |
| Excel macro-enabled | `.xlsm` | `…spreadsheetml.sheet.macroEnabled.main+xml` |
| PowerPoint macro-enabled | `.pptm` | `…presentationml.presentation.macroEnabled.main+xml` |

---

## 2. Package Structure

```
[Content_Types].xml
_rels/.rels
word/
  document.xml
  vbaProject.bin          ← OLE2 binary blob
  vbaData.xml             ← optional: Word-specific VBA supplement
  _rels/
    document.xml.rels     ← references vbaProject.bin
    vbaProject.bin.rels   ← present if vbaData.xml exists
```

### Content_Types.xml

The `.bin` extension must have a `<Default>` entry:

```xml
<Default Extension="bin" ContentType="application/vnd.ms-office.vbaProject"/>
```

The document part uses a **macro-enabled** content type:

```xml
<Override PartName="/word/document.xml"
  ContentType="application/vnd.ms-word.document.macroEnabled.main+xml"/>
```

### document.xml.rels relationship

```xml
<Relationship Id="rId2"
  Type="http://schemas.microsoft.com/office/2006/relationships/vbaProject"
  Target="vbaProject.bin"/>
```

Note the `microsoft.com` domain — this is **not** the `openxmlformats.org`
namespace used by most OOXML relationships.

---

## 3. vbaProject.bin

The file is an OLE2 (Compound File Binary) document containing:
- VBA source code streams (compressed with a proprietary RLE)
- VBA p-code (bytecode for the VBA engine)
- Project metadata (module names, references, protection)

**ooxmlsdk does not parse, validate, or modify this content.** The binary is
stored as raw bytes and round-tripped without alteration.

### OLE2 magic bytes

A valid OLE2 file begins with the 8-byte signature:

```
D0 CF 11 E0 A1 B1 1A E1
```

Processors that validate the OLE2 header will reject files that do not begin
with these bytes. For pure round-trip preservation (open → save) this is not
an issue — the original bytes are written back unchanged.

---

## 4. vbaData.xml (Word only)

`word/vbaData.xml` is an optional XML supplement used by Word to store VBA-
related data that cannot fit in the OLE2 format. It is referenced from
`word/_rels/vbaProject.bin.rels`:

```xml
<!-- word/_rels/vbaProject.bin.rels -->
<Relationship Id="rId1"
  Type="http://schemas.microsoft.com/office/2006/relationships/wordVbaData"
  Target="vbaData.xml"/>
```

Content type: `application/vnd.ms-word.vbaData+xml`

vbaData.xml is also treated as a preserve-only part. Its schema
(`VbaSuppData`) is generated by the SDK but is rarely modified directly.

## 5. SDK Access Pattern

```rust
// Open a .docm package
let package = ooxmlsdk::package::OpcPackage::from_file("doc.docm")?;
let doc_part = package.main_document_part()?;

// Access the VBA project part (if present)
if let Some(vba_part) = doc_part.vba_project_part(&package) {
  // Read the raw bytes
  let bytes: Option<&[u8]> = vba_part.data(&package);
  // Replace with new bytes (preserve scenario: no-op)
  // vba_part.set_data(&mut package, bytes.unwrap())?;
}
```

Rust struct: `VbaProjectPart` — derives `SdkPart`, no schema root type.
Binary data is accessed via the `SdkPart` trait methods `data()` /
`set_data()` / `feed_data()`.

## 6. Round-Trip Gotchas

1. **Content type must use the macro-enabled variant.** A `.docm` opened and
   saved as a regular `.docx` content type will cause Word to strip the VBA on
   next open. The content type in `[Content_Types].xml` must remain
   `macroEnabled.main+xml`, not `document.main+xml`.

2. **The `vbaProject` relationship type uses `microsoft.com`, not
   `openxmlformats.org`.** Using the wrong namespace prefix causes the part to
   be unrecognised and the VBA to disappear silently.

3. **`<Default Extension="bin">` is required.** Without it, the package is
   technically malformed. Some consumers will reject the package or treat
   `vbaProject.bin` as an unknown part.

4. **Do not re-compress or re-encode the OLE2 blob.** Any modification of the
   binary content (including reordering streams, recompressing, or changing
   padding) will corrupt the VBA. Treat the bytes as immutable.

5. **`vbaProject.bin.rels` only exists if `vbaData.xml` is present.** Do not
   create an empty `.rels` file for the VBA part if it has no relationships —
   that would add spurious markup on round-trip.

6. **Password-protected VBA projects.** The OLE2 blob may be encrypted. The
   SDK does not decrypt or validate — the bytes are still preserved correctly.
   Round-trip of an encrypted VBA project is identical to an unencrypted one.

7. **Excel `.xlsm` and PowerPoint `.pptm` do not use `vbaData.xml`.** The
   VBA supplement file is Word-specific. Excel and PowerPoint macro-enabled
   packages have only `vbaProject.bin` plus its relationship.

8. **VBA project part lives in the root of the word/ folder, not in a
   subdirectory.** The target path `vbaProject.bin` is relative to the
   document part — typically resolving to `word/vbaProject.bin`.

9. **Macro-enabled templates use distinct content types.** `.dotm` (Word
   template with macros), `.xltm` (Excel template), and `.potm` (PowerPoint
   template) each have a separate `template.macroEnabled` content type string
   rather than the `document`/`sheet`/`presentation` variant. Round-trip code
   must not canonicalise these to the non-template form, or the file will open
   as an ordinary document instead of a template.

## 7. Relationship and Content Type URIs

```
VBA project relationship type:
http://schemas.microsoft.com/office/2006/relationships/vbaProject

VBA data relationship type (Word only):
http://schemas.microsoft.com/office/2006/relationships/wordVbaData

VBA project content type:
application/vnd.ms-office.vbaProject

VBA data content type:
application/vnd.ms-word.vbaData+xml

Macro-enabled document content type (Word):
application/vnd.ms-word.document.macroEnabled.main+xml

Macro-enabled template content type (Word):
application/vnd.ms-word.template.macroEnabled.main+xml

Macro-enabled workbook content type (Excel):
application/vnd.ms-excel.sheet.macroEnabled.main+xml

Macro-enabled template content type (Excel):
application/vnd.ms-excel.template.macroEnabled.main+xml

Macro-enabled presentation content type (PowerPoint):
application/vnd.ms-powerpoint.presentation.macroEnabled.main+xml

Macro-enabled template content type (PowerPoint):
application/vnd.ms-powerpoint.template.macroEnabled.main+xml
```
